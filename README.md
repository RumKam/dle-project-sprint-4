# Нейросеть для предсказания калорийности блюд

**Задача** — создать и обучить нейросеть, которая будет предсказывать калорийность блюд по фотографии и списку ингредиентов. Модель может быть использована в fitness и health-приложениях.

**Цель** — получить DL-модель, способную оценивать калорийность блюда.

**Целевая метрика** — **MAE < 50** (Mean Absolute Error) на тестовой выборке.

## Датасет

Датасет содержит фотографии блюд, описание ингредиентов, массу порции и калорийность.

- `data/ingredients.csv`:
  - `id` — ID ингредиента
  - `ingr` — название ингредиента

- `data/dish.csv`:
  - `dish_id` — ID блюда
  - `total_calories` — целевая переменная (калорийность)
  - `total_mass` — масса блюда
  - `ingredients` — список ID ингредиентов (формат: `ingr_0000000122;ingr_0000000026;...`)
  - `split` — метка разделения на train/test

- `data/images/` — изображения блюд. Каждая подпапка названа `dish_id` и содержит файл `rgb.png`.

## Этапы проекта

### 1. EDA

#### Качество изображений и особенности съёмки
- Тарелки преимущественно квадратные, белые, но встречаются круглые разных цветов и размеров.
- Блюдо часто по центру, но продукты могут быть смещены, иногда тарелка обрезана.
- На снимках присутствуют посторонние предметы.
- Освещение варьируется: засветы, глубокие тени.
- Много мелких ингредиентов (миндаль, зелень, крупы), особенно в смешанных блюдах.

#### Распределение целевой переменной и признаков
- Калорийность (`total_calories`) имеет сильный левосторонний сдвиг.
- Большинство блюд ≤1500 ккал, редкие выбросы до 4000 ккал.
- Обнаружено аномальное блюдо с массой 3 кг и калорийностью 4000 ккал (вероятно, ошибка данных).
- Масса блюда (`total_mass`) и количество ингредиентов (`n_ingredients`) коррелируют с калориями (коэффициенты ~0.4–0.7). Добавлены как дополнительные признаки.
- Корреляционная матрица ингредиентов показала слабые связи. Наибольшее влияние у оливкового масла и соли.

### 2. Пайплайн обучения

  - `dataset.py` — загрузка данных, аугментации.
  - `utils.py` — функции обучения, валидации.

---

### 3. Обучение модели

#### Архитектура финальной модели
- **Изображения** предобученный `vit_base_patch16_224` (Vision Transformer).
- **Текст ингредиентов** предобученный `roberta-base` для получения эмбеддингов.
- **Дополнительные признаки** масса блюда и количество ингредиентов конкатенируются к объединённому эмбеддингу.
- **Регрессионная голова** два полносвязных слоя с Dropout.

#### Процесс обучения
- Валидационная выборка выделена из train (20%) случайным образом.
- Лучший результат на **26-й эпохе**:
  - Train MAE: **33.74**
  - Validation MAE: **30.17**
  - Validation Loss: **0.05**
- После 25-й эпохи loss вышел на плато.
- Переобучения не наблюдалось.

### 4. Валидация качества на тестовом наборе

Инференс на тестовом сплите дал итоговую **MAE = 31.2** (цель <50 ккал достигнута).

### Анализ сложных случаев
- **Мелкие ингредиенты с высокой калорийностью** в двух блюдах присутствует миндаль. Модель недооценивает калории, так как мелкие орехи трудно различимы на изображении.
- **Смешанные блюда** ингредиенты визуально перекрывают друг друга, затрудняя идентификацию и оценку количества.
- **Сложность композиции** во всех пяти примерах несколько ингредиентов, их визуальные признаки схожи, что приводит к путанице.

## Результаты

Разработанная гибридная модель на основе **ViT-base-patch16-224 + RoBERTa-base** с дополнительными признаками достигла **MAE = 31.2** на тестовом наборе, что значительно ниже целевого порога 50 ккал.
